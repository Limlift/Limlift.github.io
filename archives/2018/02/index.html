借力「数据」, 我的「弱自律」2017
  • 总结

前言经常逛知乎，其实非常敬佩艳羡那些自律始终如一的人。作为一名UI设计师，我能够严谨规范地做好我的设计工作，让我的图层/设计稿/规范文档都井然有序地排排坐，但是对于生活而言，我却无法如此得心应手。我也曾照猫画虎他人的「自律计划」，幻想计划过后诞生一个「高度自律」的优秀的我（笑），事实却是我很难对计划持之以恒，幻想终究存在于幻想之中。 网上有个段子，说是每到新年/春节前后，总会有人制定一堆「梦想型」新年计划，等着下一年年末拿这些计划打自己的脸。很多时候我们没法完成新年计划，大多数都是因为两个原因，一是计划制定得不切实际，一是没法在执行中接受实时反馈失去动力。17年初的时候我就在想，今年可不要活得像个段子一样，针对上面两个「计划阻碍者」，我决定借力「数据」的力量，并结合自己的特点，制定了一个「弱自律」的新年计划。 所谓「弱自律」，就是字面上的意思，即尽量去做而不是强求，并非一定要强行达到，量力而行。我的2017计划就是减脂（稳定体脂率低于16%）和生活数据化（让自己的生活有迹可循）。17年一年过去，翻阅记录，发现我竟然神奇地完成了计划，体脂率降到了15%，全年生活有记录可查。 减脂多说无益，我就直接上数据吧，以下是我17年初到18年初的测量数据记录，用的是「好轻」的体脂称和配套APP。 事实上这个过程单纯从数据上看很光鲜靓丽，其实数据后面的计划真的是「弱自律」化到了极致。 减脂的计划具体就是「无器械健身」+「节奏饮食」。 无器械健身「无器械健身」指的是不用专业器材的健身，我使用了「Keep]作为指导以及辅助。作为非专业人士我没法去判定「Keep」的专业型如何，但使用了一年下来，我发现它确实是一款实用好使的辅助应用，尤其是在数据记录以及反馈这一方面。 从我的「Keep」2017年度报告上看，我的确因为或这或那的原因中断过健身，尤其是坚持健身一段长时间后的那种疲惫，你可以看到我的中止基本上是周期性的，之所以能够重新出发，主要还是因为定时测量的「好轻」体重体脂率数据以及「Keep」的训练数据实时反馈，把我从舒适区里拉了出来。 另一个让我脱离舒适区的，就是数据本身的趣味性。「Keep」把一些里程碑数据给可视化成了我们游戏中常见的成就系统类似物-「徽章」元素，只要达到某种程度的里程碑就可以获取对应的「徽章」。在下半年的更新里面还推出了根据训练数据给定的奖励-「定制化服务」，这种数据的趣味性也对我产生了动力性的作用，让我会以一种玩游戏收集的态度去执行计划。 节奏饮食「节奏饮食」指的是规律科学的饮食，这方面我只是 「Keep」根据不同的人群以及不同的健身需求，推出了精确分类分级的健身指导教程，以K1~K5的级别囊括了所有健身人群，以人体部位分「胸/背/肩/臂/颈/腹/腰/臀/腿/全身」，以目标分「减脂/增肌/塑形/瑜伽」，以职业分「上班族/学生/家庭主妇/驾驶员」，以场景分「户外/健身房」等等，分类以及定位齐全精确，照顾了以「健身」为目的的大目标人群的同时，还精细地为细分类人群定制服务。 「Keep」的记录功能基本是我亲自看着变更过来的，从一开始简单地记录用户对于「健身/跑步」的时间统计和「跑步」详情记录（包含路线/数据），到现在的用户年度健身报告/「骑行」详情记录以及用户等级徽章的大版本变更，都体现了倾向用户体验的迭代趋势。 生活数据化当下的技术离真正意义上的「数字化自我/量化自我」其实还很远，真正要把自己数字化，需要测量的数据太多了。因此我定下的目标仅仅是「让自己的生活有迹可循」，只是做生活上的简单数据记录。我使用了「印象笔记」作为辅助，事实上「印象笔记」也一直是我的知识库管理工具。 「一千个读者就有一千个哈姆雷特」，不同的人对「印象笔记」的使用习惯也不一样，其实只要符合自己的特点/行事习惯，最终能够达到目的即可。 工作数据对于我的工作而言，数据的记录是一直存在的。这是我工作伊始就养成的好习惯，能够让我清晰地查阅过去的工作从而做出总结调整，同时可以免背一些锅，毕竟这个行业里背锅是常态… 对于工作的数据，我记录的格式是「项目/产品名称」+「工作内容」+「时间范围」，尽量简短地写明内容。 生活数据对于生活的数据，我记录的格式是「生活内容」+「完成日期」，并且严格按照月份区分的形式分组。 对于生活数据记录方面，更重要的其实是目的性和随手性。很多生活上的事情，都是突发的，我一般都把生活上的事务按照作为一个待办事项记录。 对于本来在这个月完成的事务，本月没有完成需要顺延的时候，我会作保留「未完成」待办事项的处理，然后再把该待办事项复制到下一个月，这样我就能清楚该事项持（tuo）续（yan）了多久，有一个直观的认识。 书影数据我很喜欢看书看电影，大多数时候还会写观后感或者影评。我还会对书籍封面/电影海报进行二次加工做成题图，来搭配对应的观后感和影评。因此我也会对看过的书籍和电影做一个数据的记录。基本记录格式是「书籍/电影名称」+「出版/上映日期」+「阅读/观看日期」。 后记所谓「自律」与「记录」，重点不在于工具和本身所包含的含义，而是需要「持之以恒」，再好的工具遇到懒惰二字也很难克服。

对于喜欢折腾个人博客的人来说，一个可玩性强的博客/博客平台才是最适合自己的。可玩性包括自定义主题/功能，现有的通用型博客平台在自定义这一块上基本没法满足可玩性，太专业的博客系统对于非专业型人才来说门槛又比较高，并且管理繁杂。因此这里推荐一种比较简单的搭建个人博客方式：Hexo+GitHub Pages。 对于0技术基础并且无心折腾的人或者是只专注文字方面的作者来说，本教程一般不适合你，推荐使用通用型的博客平台-「简书」 Hexo和GitHub Pages以及个人博客搭建的概念Hexo 是一款快速/简洁/高效的博客框架，依赖于Node.js，拥有丰富的插件定制，并且支持一键发布到平台。它使用Markdown（或其他渲染引擎）解析文章，可以快速生成静态网页文件从而形成博客。 GitHub Pages 是GitHub推出面向GitHub用户的一个脱管静态网页的功能，允许用户在GitHub的服务器上上传自己的静态网页文件并提供访问浏览，以供用户做说明文档/博客等使用。 个人博客搭建，一般可以拆解为三个步骤：你需要去选择一个合适的博客系统，把系统上传部署到你所购买的服务器上，然后在服务器上设置好你购买的域名。 这样下来包含的时间成本（学习博客系统/服务器）以及金钱成本（购买服务器/域名）会比较高，而使用Hexo+GitHub Pages的方式去搭建博客，相当于GitHub帮助你购买了域名和服务器，省去了这两块成本，Hexo的简单上手性又减少了你学习博客系统的时间成本。 环境与前期准备 安装Node.js 到Node.js官网 下载对应平台的版本，推荐选择LTS稳定版，安装。 安装Git 到Git官网 下载对应平台的版本，安装。 安装文本编辑器 下载你喜欢的文本编辑器，方便后面的文本/代码编辑，代码编辑器推荐Sublime Text 和 Visual Code ，Markdown文本编辑器推荐 Typora ，安装。 GitHub帐号与Pages（有帐号以及pages的可跳过） 注册/登录Github帐号，新建一个「仓库（Repository）」，命名为「你的GitHub账户名.github.io」 Markdown Markdown基本语法 入门，学会md文档写作有助于博客发布 GitHub Pages的部署 打开Git，输入GitHub的邮箱和用户名： 12git config --global user.email &quot;yourname@gmail.com&quot;git config --global user.name &quot;yourname&quot; 盛名之下，其实难副
  • 影评

如我这种只读过福尔摩斯探案集却不怎么看过东野圭吾/阿加莎作品的人自然没有办法称作为侦探迷，我只是对刑侦作品中表现出来的逻辑性/戏剧性感兴趣，在完全没有接触过任何《东方快车谋杀案》相关的作品（包括小说/前几个版本的电影/书评/致敬）的情况下，我在春假期间补完了这部2017年底的翻拍作品。 即使是我这种没有读过原作小说系列的人，我都能感受到这部作品是令人期待的，因为上映前口碑/营销以及风评无一不是在表达这种一致的情感。原作的名声自然大，普通人都应该略有耳闻，我也抱着拜读的心情想从这部17年翻拍作中一窥原作，甚至是领略一下原作的魅力。 始料未及的是，完整的电影看完下来，留给我的只有一句深深的“盛名之下其实难副”的印象。 暂且放下这种印象产生的原因探究，这部电影作品在视觉上带给人的享受绝对是一流的，虽然免不得有当代先进影像技术的加成，但是镜头以及后期的处理还是给我们真实地呈现了作品案件发生背景-雪山因故停靠的列车/案件的人物-因某种原因齐聚的包含贵族大亨仆从医生各阶层的人物/案件本身的进行过程。除了纷飞逼真的雪花雪崩，电影中的人物定妆以及服装也是下了大功夫的，里里外外透着一股精致感。影片最后的结局揭幕场景也非常的令人震感，布景完全是致敬了达芬奇的《最后的晚餐》，非常的巧妙。 可惜的是这种唯美的视觉衬不上僵硬的剧情，就如同最后致敬宗教式的致敬完全不符合思想的表达一样，割裂感太过于严重。 如果说这部电影真的能够完全阐述《东方快车谋杀案》原作，要么看过原作的都是阿炳，要么原作本身就是有问题的。从现实的角度去考量，这两种可能性都不怎么可能存在，那么只有第三种可能性-电影拍烂了。 记得有一个影评人说过，特效技术上的烂永远比不过剧情剧本上的烂，坏在面子永远比不过坏在里子。可能你会反驳，既然阿加沙的著作做内核，再怎么拍也不会烂到哪里去。诚然，影片观赏到最后，对于案件所揭示的真相以及思想自然是能让人倒吸一口冷气并且为之深思的，但是影片本身的类型是侦探悬疑，一部侦探悬疑作品，由头到尾都在故弄玄虚，推理完全无法让观众参与，只留下一个的确令人深思的结局，不得不说让人很失望。 对于我这种没有阅读过原作甚至其他阿加沙作品的非粉丝情怀观众来说，剧情才是最重要的。电影对于推理的描写实在是无法让我代入，从证据的考究以及思考推敲的过程，根本不像是符合逻辑的思考，反而像是灵机一动或者灵感的闪现，人物之间的关系推理全在主角波洛的一张嘴，没有足够的客观证据细节支持，让人觉得电影的推理实在是太主观臆断了，像是这部电影就是为了塑造波洛这个新福尔摩斯侦探形象而塑造的。 人物的塑造也不得不把我推向上述的结论，电影中出场的人物似乎都缺乏一种灵性，要说有什么特点那也只有「死气沉沉」了。所有的人物都绷着一张童年丧亲中年丧妻晚年丧子的严肃脸，仿佛世界上再也没有什么快乐能让这一车子人笑起来了，就算是案件询问中欢快的马奎斯，在询问到关键点也迅速地卸下了欢快的伪装露出了极为僵硬的沉重。所有的人物特质在这种压抑的气氛中都被冲淡了，要辨认这些人物我也只能通过面容和服装，我甚至很难去记清每个人物的职业背景，况且电影和剧本也没有花多少镜头去描写人物的特质，当然沉迷于狄更斯小说呵呵直乐的波洛算是例外，这就使得电影本身的推理体验性更差了。 结局诚然让人震撼，但其实《最后的晚餐》构景似乎没有什么必要。硬要将剧情跟这幅宗教画作扯在一起，大概就是人物的关系和画作的人物关系比较一致吧，但并没有什么宗教寓意在里面啊！ 我需要去阅读原作来冲淡此时我给《东方快车谋杀案》留下的坏印象，同时我又希望导演们不要再糟蹋这些经典了。
